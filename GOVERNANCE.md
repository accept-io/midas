# Governance

## Overview

Accept MIDAS is an early-stage open-source project currently operating under a BDFL (Benevolent Dictator For Life) model. As the project and its contributor community grow, governance will evolve toward a maintainer committee model.

This document describes how decisions are made, how maintainers are recognised, and how this document itself may change.

## Decision-Making

**Day-to-day decisions** — routine changes (bug fixes, documentation improvements, minor enhancements) are made by maintainers through pull request review and merge. A single maintainer approval is sufficient.

**Significant changes** — API changes, architectural direction, new external dependencies, and changes to governance require a GitHub issue for discussion before implementation. Maintainers should reach consensus through that issue thread before proceeding.

**Disputes** — during the current BDFL phase, the founding maintainer retains final decision authority when consensus cannot be reached.

## Becoming a Maintainer

MIDAS welcomes contributors who want to take on a deeper role. Maintainership is based on sustained contribution and demonstrated understanding of the project, not seniority or volume of commits.

The process:

1. A contributor makes sustained, high-quality contributions over time.
2. They demonstrate an understanding of the project architecture, goals, and governance principles.
3. An existing maintainer nominates them by opening a pull request adding their name to the maintainer table above.
4. If no existing maintainer raises an objection within 14 days, the nomination is accepted and the pull request is merged.

Maintainers are expected to:

- Review and merge pull requests in a timely manner.
- Participate in issue triage and community questions.
- Uphold the Code of Conduct in all project spaces.
- Keep the project aligned with its governance principles and CNCF requirements.

Maintainers who are inactive for an extended period may be moved to emeritus status by mutual agreement.

## Changing This Document

Changes to this governance document require a pull request and explicit approval from the founding maintainer. During the current BDFL phase, the founding maintainer retains final authority over governance changes.

As the project matures and the maintainer group grows, this document will be updated to reflect a more distributed decision-making model.

## CNCF Alignment

Accept MIDAS is a CNCF Sandbox project candidate. The project operates in accordance with CNCF principles and community expectations.

The project adopts the CNCF Code of Conduct. See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

All governance decisions are made transparently through public GitHub issues and pull requests.
